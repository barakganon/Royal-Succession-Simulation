## What went well
- **Live "run it for real" caught a bug unit tests missed.** 12-1's unit tests passed (clean data), but a live smoke against the dev DB threw `TypeError` — `HistoryLogEntryDB.year` is nullable (system messages), breaking the year comparison. Fixed (skip year-less events) + regression test. This is the Epic-3-retro lesson paying off again: green ≠ correct on real data.
- **Authed live render check for the visual story (12-3).** Didn't trust presence-tests alone — logged in with a cookie jar and confirmed the book route renders 23KB HTML with real chapter content and the PDF route returns a valid 5.3KB `%PDF`. Per the Epic 3 retro process change.
- **The "no worktree for sequential" policy (Epic 11) worked perfectly.** All three Sonnet subagents ran on live `main` with zero staleness/divergence — the exact failure mode 11-3 hit is gone. Each built cleanly on the prior story's merged work.
- **Spec guardrails landed.** The 12-1 spec's #1 warning (chapters from `reign_start_year IS NOT NULL`, NOT `is_monarch`) was respected by the agent — the multi-generation bug never happened.
- **Right-sized risk on PDF.** Chose reportlab (pip-only, the pdf skill's tool) over weasyprint (system libs) → fully verifiable headlessly. SVG-in-PDF cut as documented scope, not a surprise.

Caught by the integrator (dry-run resolve) and fixed. Tests didn't catch it (reportlab was already installed in the venv). ⚑ Action item.
