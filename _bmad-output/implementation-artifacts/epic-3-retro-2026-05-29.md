## What went well

1. **Worktree contract-first parallelism.** Each multi-agent story partitioned by *file region/concern* (script vs markup vs CSS vs tests) with a frozen interface contract. Three multi-agent stories, **zero merge conflicts** — the partition, not luck.
2. **Contract-first tests as a coherence check.** The test agent's assertions failed in isolation and went green on integration — proving the blind agents built the same thing. This caught a real drift in 3-5 (a test asserting a non-existent summary key) at integration rather than later.
3. **Detailed specs kept agents on-rails.** Current-code maps with line numbers + explicit out-of-scope sections meant agents reused existing systems (GeoJSON, IIFE) instead of reinventing them.
4. **Test discipline held.** Suite grew 277→315 and was green at every merge; deferred findings were logged in `deferred-work.md` rather than silently dropped.

## What didn't go well

1. **🔴 Visual verification was deferred every single story — and the map shipped blank on real data.** Found only by running the app *during this retro*: `map_renderer` emits raw pixel coords as `col/row`, but the canvas `hexCenter` treated them as hex-grid indices (×48 off-screen), so `/world/map` rendered an empty canvas against a real generated game. Epic 3 was marked "done" with its central screen non-functional.
2. **Tests-green created false confidence.** Every Epic 3 test asserted code/markup *presence* (substring checks) — none rendered the map with real data or checked pixels. The whole epic was "green" while the core feature was broken.
3. **Orchestration friction.** (a) Spawned worktree agents defaulted to *plan mode* (3-4) and returned plans instead of code — wasted a spawn round until re-prompted with an explicit "execute now, pre-approved" directive. (b) Worktrees branched off `main`, not the feature branch holding the freshly-committed spec, so the story file was absent in agents' trees (mitigated only because the contract was inlined in each prompt).
4. **A test-infra footgun masked a data problem.** Running `pytest` silently wiped the dev DB (see below), so the dev environment kept coming back empty — which is part of why nobody hit the blank-map issue manually earlier.

## Key insights / lessons

- **Tests-green ≠ done for a visual feature. Run the actual screen.** Substring/DOM-presence tests verify wiring, not rendering. A visual feature needs a real run-the-app check (a headless-Chrome screenshot is cheap) before "done."
- **Single-file stories parallelize by *region + contract*, not by feature.** 3-4 was almost all `world_map.html`; the win came from splitting script/markup/css/tests with a frozen contract.
- **The contract-first test agent is both the linchpin and the weak point** — it proves coherence but is the most likely source of drift ("bonus" assertions outside the contract are where it creeps in). Keep test assertions quoting the contract verbatim.
- **Deferred verification compounds.** Three stories each deferring the visual check turned into a wholly non-functional core screen discovered an epic later.
